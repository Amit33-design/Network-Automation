Usage Guide

1. Install dependencies

pip install netmiko

2. Set credentials (never hard-code)

export NET_USER=admin
export NET_PASS=yourpassword
export NET_ENABLE=enablepassword   # if required

3. Run pre-check BEFORE deployment

python3 pre_check_<site>.py
# Saves: pre_baseline_<site>.json

4. Deploy your configs

5. Run post-check AFTER deployment

python3 post_check_<site>.py
# Reads: pre_baseline_<site>.json
# Saves: post_report_<site>.json
# Alerts: BGP peer drop, route count drop >5%, interface errors spike

Exit codes: 0 = all OK · 1 = unreachable devices · 2 = post-check alerts

Management IPs default to 192.168.1.X — edit DEVICES list in the generated script to match your out-of-band addressing.

Deploy Guide

Prometheus alert rules

# Copy to Prometheus rules directory
cp alerts-<site>.yml /etc/prometheus/rules/
# Reload Prometheus
curl -X POST http://localhost:9090/-/reload

Grafana dashboard (API import)

curl -X POST http://admin:admin@localhost:3000/api/dashboards/import \
  -H 'Content-Type: application/json' \
  -d @dashboard-<site>.json

SNMP exporter targets — add your device IPs to snmp.yml modules, then scrape via:

- job_name: network_devices
  static_configs:
    - targets: [<device-mgmt-ips>]
  metrics_path: /snmp
  params:
    module: [if_mib, bgp]
  relabel_configs:
    - source_labels: [__address__]
      target_label: __param_target
    - target_label: __address__
      replacement: snmp-exporter:9116

Datasource UID in the dashboard JSON is prometheus — rename to match your Grafana datasource UID if different.
