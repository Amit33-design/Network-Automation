Day-0 vs Day-N Separation

Day-0 (this tab) = management plane only: hostname, mgmt IP, SSH v2, NTP, syslog, LLDP, ZTP callback URL. No BGP, no VLANs, no VXLAN.

State Board

Per-Device ZTP State Machine

States update automatically in production via POST /api/ztp/callback from each device.
Use → Next to manually advance state during testing. ↺ resets to REGISTERED.

REGISTERED → POWERED_ON → DHCP_ACK → SCRIPT_DOWNLOADED → CONFIG_APPLYING → CALLBACK_RECEIVED → VERIFIED → ONLINE

Complete Step 1 and click Generate ZTP to initialize state board.

ZTP API

ZTP State Machine API

Flask API that tracks per-device ZTP state. Devices POST to /api/ztp/callback at each provisioning milestone. Runs as a Docker sidecar (see File Server tab).

MethodEndpointDescription
POST/api/ztp/registerPre-register a device
POST/api/ztp/callbackDevice reports state change
GET/api/ztp/stateAll device states (JSON array)
GET/api/ztp/state/<hostname>Single device state

Batfish Dry-Run

Batfish Semantic Validation

Validates all generated device configs semantically — before any SSH connection — using Batfish network analysis. Catches configuration errors that pre-checks miss.

CheckWhat it catches
Undefined referencesRoute-maps, prefix-lists, ACLs referenced but not defined
BGP session reachabilityPeers unreachable in Batfish's data-plane model
Loopback route coverageMissing routes to any device loopback in the fabric
 
# Start Batfish (one-time):
docker run -d -p 9997:9997 -p 9996:9996 batfish/batfish

# Run validation:
pip install pybatfish pandas
python3 batfish_validate_<site>.py

# Exit codes: 0 = all passed · 1 = failures · 2 = Batfish error

Usage Guide

Usage Guide

1. Install dependencies

pip install netmiko

2. Set credentials (never hard-code)

export NET_USER=admin
export NET_PASS=yourpassword
export NET_ENABLE=enablepassword   # if required

3. Run pre-check BEFORE deployment

python3 pre_check_<site>.py
# Saves: pre_baseline_<site>.json

4. Deploy your configs

5. Run post-check AFTER deployment

python3 post_check_<site>.py
# Reads: pre_baseline_<site>.json
# Saves: post_report_<site>.json
# Alerts: BGP peer drop, route count drop >5%, interface errors spike

Exit codes: 0 = all OK · 1 = unreachable devices · 2 = post-check alerts

Management IPs default to 192.168.1.X — edit DEVICES list in the generated script to match your out-of-band addressing.

Deploy Guide

Deploy Guide

Prometheus alert rules

# Copy to Prometheus rules directory
cp alerts-<site>.yml /etc/prometheus/rules/
# Reload Prometheus
curl -X POST http://localhost:9090/-/reload

Grafana dashboard (API import)

curl -X POST http://admin:admin@localhost:3000/api/dashboards/import \
  -H 'Content-Type: application/json' \
  -d @dashboard-<site>.json

SNMP exporter targets — add your device IPs to snmp.yml modules, then scrape via:

- job_name: network_devices
  static_configs:
    - targets: [<device-mgmt-ips>]
  metrics_path: /snmp
  params:
    module: [if_mib, bgp]
  relabel_configs:
    - source_labels: [__address__]
      target_label: __param_target
    - target_label: __address__
      replacement: snmp-exporter:9116

Datasource UID in the dashboard JSON is prometheus — rename to match your Grafana datasource UID if different.
